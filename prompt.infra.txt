Leia o arquivo PLANS.infra.md na raiz do projeto.
Este documento é um documento vivo de infraestrutura. Sua função é:

1. Ler e entender todo o conteúdo do PLANS.infra.md
2. Executar as tarefas do checklist em ordem, começando pelo P-DEPLOY-1
3. A cada tarefa concluída, marcar o item como [x] no próprio PLANS.infra.md
4. Atualizar o campo Última atualização da seção correspondente com a data e hora atual
5. Após concluir cada fase (P-DEPLOY-1, P-DEPLOY-2, etc.), fazer um commit com a mensagem no formato: infra: conclui P-DEPLOY-1 — preparar projeto para produção
6. Nunca pular uma tarefa sem registrar o motivo no documento
7. Se encontrar um bloqueio, registrar em ## Observações e Riscos com data e descrição

Regras obrigatórias:

- Nunca expor META_ACCESS_TOKEN em nenhum arquivo versionado
- Portas do Campaign Builder: backend 3001, PostgreSQL 5433, Nginx 8081 — nunca usar 3000/5432/8080 (reservadas para o Bikkoo)
- Frontend React exige npm run build — nunca servir o Vite diretamente em produção
- O VITE_BACKEND_URL deve sempre apontar para HTTPS
